# CrediShoping — Sistema de Ventas a Crédito

## Descripción del Proyecto
Aplicación fullstack SaaS multi-tenant para gestión de ventas a crédito. Permite a múltiples empresas (tenants) registrar clientes, productos, ventas en cuotas, pagos, compras de inventario y auditoría de operaciones — cada empresa ve únicamente sus propios datos.

## Stack Tecnológico

### Backend
- **Runtime**: Node.js con `"type": "module"` (ES Modules — usar `import/export`, nunca `require`)
- **Framework**: Express 5.x
- **Base de datos**: PostgreSQL 16 via librería `pg` (connection pool)
- **Auth**: `jsonwebtoken` + `bcryptjs`
- **Variables de entorno**: `dotenv` (.env en `/server/.env`)
- **Dev**: `nodemon`

### Frontend
- **Framework**: React 19 con Vite 7
- **Routing**: React Router DOM 7
- **UI**: Bootstrap 5.3 + Bootstrap Icons
- **HTTP Client**: Axios (header Authorization inyectado automáticamente por AuthContext)
- **Gráficos**: Recharts (gráfico de barras en Dashboard)
- **Estilos**: CSS custom properties en `App.css` + clases Bootstrap

### Base de Datos
- **Motor**: PostgreSQL
- **Nombre de DB**: `kreditfy`
- **Tablas principales**: `tenants`, `customers`, `products`, `admins`, `sales`, `sale_details`, `payments`, `shopping`, `audit_logs`, `monthly_closings`, `exchange_rates`
- **Triggers**: `increase_stock` (INSERT en shopping), `decrease_stock` (BEFORE INSERT en sale_details)

## Arquitectura Multi-tenant

El sistema es multi-tenant: múltiples empresas comparten el mismo servidor y base de datos pero cada una ve solo sus datos. La separación se hace mediante `tenant_id` en todas las tablas de negocio.

### Roles del sistema

| Rol | Descripción | `tenant_id` en JWT |
|-----|-------------|---------------------|
| `platform_admin` | Administra la plataforma, crea y gestiona tenants | `null` |
| `superadmin` | Administra un tenant específico, gestiona admins | ID del tenant |
| `admin` | Usuario operativo de un tenant | ID del tenant |

### Flujo de autenticación

1. `POST /auth/login` → verifica credenciales, emite JWT (8h) + refresh token (7d)
2. JWT payload incluye: `id`, `name`, `email`, `role`, `permissions`, `tenant_id`, `tenant_name`
3. Todas las rutas de negocio aplican: `authenticateToken` → `resolveTenant`
4. `resolveTenant` pone `req.tenantId` (null si es platform_admin)
5. Cada modelo filtra con `AND tenant_id = $N` cuando `tenantId != null`; sin filtro si es null (platform_admin ve todo)

### Rutas según rol

- `platform_admin` → login redirige a `/platform`, accede a `/platform/*`
- `superadmin` / `admin` → login redirige a `/`, accede a las rutas normales del tenant

### Endpoints de plataforma (`/platform/*`)

| Método | Ruta | Descripción |
|--------|------|-------------|
| `GET`  | `/platform/stats` | Métricas globales (tenants, ventas, clientes) |
| `GET`  | `/platform/tenants` | Listar todos los tenants |
| `POST` | `/platform/tenants` | Crear tenant |
| `GET`  | `/platform/tenants/:id` | Detalle de un tenant |
| `PUT`  | `/platform/tenants/:id` | Editar tenant (nombre, slug, moneda, activo) |
| `GET`  | `/platform/tenants/:id/admins` | Admins del tenant |
| `POST` | `/platform/tenants/:id/admins` | Crear superadmin/admin para un tenant |

## Convenciones de Código

### Backend

**Patrón de rutas** (`routes/`):
```js
import { Router } from "express";
import { authenticateToken, requireSuperAdmin } from "../src/middleware/authMiddleware.js";
import { resolveTenant } from "../src/middleware/resolveTenant.js";
import { getX, createX } from "../src/controllers/xController.js";

const router = Router();
router.get("/x", authenticateToken, resolveTenant, getX);
router.post("/x", authenticateToken, resolveTenant, createX);
export default router;
```

**Controladores** (`src/controllers/`):
```js
import * as xModel from "../models/xModel.js";
import { createAuditLog } from "../models/auditModel.js";

export const getX = async (req, res) => {
  try {
    const data = await xModel.getAllX(req.tenantId);
    res.json(data);
  } catch (error) {
    console.error(error);
    res.status(500).json({ error: "Error al obtener X" });
  }
};

export const createX = async (req, res) => {
  try {
    const { field1, field2 } = req.body;
    if (!field1) return res.status(400).json({ error: "field1 es obligatorio" });
    const result = await xModel.createX(req.body, req.tenantId);
    await createAuditLog({
      admin_id: req.admin.id, action: 'CREATE', table_name: 'x',
      record_id: result.id, description: `Creó X: ${result.field1}`,
      tenant_id: req.tenantId
    });
    res.status(201).json(result);
  } catch (error) {
    console.error(error);
    if (error.code === "23505") return res.status(400).json({ error: "Ya existe" });
    res.status(500).json({ error: "Error al crear X" });
  }
};
```

**Modelos** (`src/models/`) — patrón multi-tenant:
```js
import pool from "../../db/config.js";

export const getAllX = async (tenantId) => {
  const conditions = [];
  const params = [];
  let idx = 1;
  if (tenantId != null) { conditions.push(`tenant_id = $${idx++}`); params.push(tenantId); }
  const where = conditions.length ? `WHERE ${conditions.join(' AND ')}` : '';
  const result = await pool.query(`SELECT * FROM x ${where} ORDER BY id`, params);
  return result.rows;
};

export const createX = async (data, tenantId) => {
  const { field1, field2 } = data;
  const result = await pool.query(
    `INSERT INTO x (field1, field2, tenant_id) VALUES ($1, $2, $3) RETURNING *`,
    [field1, field2, tenantId]
  );
  return result.rows[0];
};

export const updateX = async (id, data, tenantId) => {
  const result = await pool.query(
    `UPDATE x SET field1 = $1 WHERE id = $2 AND tenant_id = $3 RETURNING *`,
    [data.field1, id, tenantId]
  );
  return result.rows[0];
};

export const deleteX = async (id, tenantId) => {
  await pool.query(`DELETE FROM x WHERE id = $1 AND tenant_id = $2`, [id, tenantId]);
};
```

**Transacciones** (para operaciones multi-tabla):
```js
const client = await pool.connect();
try {
  await client.query("BEGIN");
  // ... operaciones ...
  await client.query("COMMIT");
  return result;
} catch (error) {
  await client.query("ROLLBACK");
  throw error;
} finally {
  client.release();
}
```

**Formularios** (`src/components/`):
```jsx
import { useState } from 'react';
import axios from 'axios';

const FormX = ({ onClose, onSaved }) => {
  const [form, setForm] = useState({ field1: '', field2: '' });
  const [error, setError] = useState('');

  const handleChange = (e) => setForm({ ...form, [e.target.name]: e.target.value });

  const handleSubmit = async () => {
    try {
      const res = await axios.post('http://localhost:3000/x', form);
      onSaved?.(res.data);
      onClose?.();
    } catch (err) {
      setError(err.response?.data?.error || 'Error al guardar');
    }
  };

  return (
    <div className='border border-secondary-subtle rounded p-4'>
      <div className='d-flex justify-content-between'>
        <h3>Nuevo X</h3>
        <button type="button" className="btn-close" onClick={onClose} aria-label="Close"></button>
      </div>
      {error && <div className="alert alert-danger">{error}</div>}
      <div className="row g-3">
        <div className="col-md-12 text-start">
          <label htmlFor="field1" className="form-label">Campo 1</label>
          <input
            type="text"
            className="form-control"
            id="field1"
            name="field1"
            value={form.field1}
            onChange={handleChange}
          />
        </div>
        <div className="col-md-12 text-start">
          <button type="button" className="btn btn-success" onClick={handleSubmit}>Guardar</button>
          <button type="button" className="btn btn-danger ms-3" onClick={onClose}>Cancelar</button>
        </div>
      </div>
    </div>
  );
};

export default FormX;
```

## Reglas de Estilo Importantes

- **ES Modules siempre**: `import/export`, nunca `require/module.exports`
- **Arrow functions** para componentes y handlers
- **Nombres de archivos**: PascalCase para componentes React, camelCase para todo lo demás
- **Importaciones backend**: usar rutas con extensión `.js`
- **JSX**: usar `className` (no `class`), `htmlFor` (no `for`)
- **Sin estilos inline**: nunca usar `style={{}}` en JSX — definir clases en `App.css`
- **Formularios multi-paso**: usar `<div>`, nunca `<form>` — evita submit accidental con Enter
- **Formularios simples**: pueden usar `<form onSubmit>` si son de un solo paso
- **Botón cerrar formulario**: pasar `onClose` como prop
- **Mensajes de error del servidor**: en español, descriptivos
- **SQL**: parámetros posicionales `$1, $2...`, siempre `RETURNING *` en INSERT/UPDATE
- **CSS**: usar variables CSS definidas en `App.css` (`var(--primary-color)`, etc.)
- **Auditoría**: todo CREATE/UPDATE/DELETE debe registrar un log con `createAuditLog`
